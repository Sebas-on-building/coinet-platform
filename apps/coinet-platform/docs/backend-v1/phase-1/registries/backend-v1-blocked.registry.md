## B. Version-Sprawl Blocks (Plan 1.5)

### Prohibited Implementation Naming Patterns

```text
-v2, -v3, -v4, -vN     (new implementation variants)
-next, -new            (replacement ambiguity)
-final                 (false finality)
-complete              (false completeness)
-master                (ambiguous supremacy)
-comprehensive         (marketing-like)
-ultimate              (non-engineering)
-rebuilt               (hidden rewrite fork)
```

### Protected Capability Families (PSC-001..PSC-010)

```text
PSC-001 SCORING
PSC-002 DERIVATIVES_INTELLIGENCE
PSC-003 NEWS_INTELLIGENCE
PSC-004 SOCIAL_INTELLIGENCE
PSC-005 SENTIMENT_INTELLIGENCE
PSC-006 PROVIDER_DATA_FETCHING
PSC-007 ANOMALY_MONITORING
PSC-008 JUDGMENT_ORCHESTRATION
PSC-009 AI_RESPONSE_GENERATION_OR_FORMATTING
PSC-010 ALERT_INTELLIGENCE_OR_ROUTING
```

New parallel services in any PSC-NNN domain are blocked.

### Violation Classes (VSV-A..J)

prohibited version suffix, prohibited final/complete suffix, new parallel service without canonical owner, new alternative path without deprecation/replacement, new shadow without expiration, new duplicated capability with unclear product meaning, naming pattern that hides architectural decision, implementation split with no Phase 1–3 justification, parallel feature created because existing service is messy, legacy variant kept active after replacement decision.

### Exception Procedures

| Block type          | Procedure                  |
| ------------------- | -------------------------- |
| Architecture freeze | AFE (Plan 1.4)             |
| Version sprawl      | FRP / BSCP / VSE (Plan 1.5) |
