# Phase 3 BTAR Records

This folder holds Phase 3 Backend Task Admission Records (P3-BTAR-NNN).

## Naming Convention

```text
P3-BTAR-NNN-<kebab-case-title>.md
```

Examples:

```text
P3-BTAR-001-synthetic-episode-contract-and-fixture-system.md
P3-BTAR-002-judgment-truth-runner.md
P3-BTAR-003-synthetic-episode-corpus.md
P3-BTAR-004-semantic-assertion-engine.md
P3-BTAR-005-full-truth-suite-execution-and-report.md
P3-BTAR-006-serious-judgment-flaw-remediation.md   (conditional, only if P3-BTAR-005 surfaces serious flaws)
```

## Admission Process

Every P3-BTAR is admitted via the Plan 1.6 process (BTAR template + eight-question gate + scope mapping). Plan 3.0 does not short-circuit Plan 1.6.

## Indexing

Every record in this folder MUST appear in:

1. `phase-3/registries/phase-3-btar.registry.md` (Phase 3 BTAR view).
2. `phase-3/registries/phase-3-record-index.registry.md` (Phase 3 master index).
3. `phase-1/registries/backend-v1-record-index.registry.md` (cross-phase master index).
4. (On completion) `phase-1/registries/backend-v1-decision-log.registry.md`.

## Authority

Plan 3.0 §9 defines the expected P3-BTAR sequence and what each BTAR is required to deliver.
